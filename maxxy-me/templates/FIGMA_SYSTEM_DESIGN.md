# Figma System Design

This file must be named `FIGMA_SYSTEM_DESIGN.md` when created at the project
root for a Figma implementation task.

Write or update it after Figma MCP extraction and before implementation code
starts. It must be based on MCP evidence, screenshots, variables, and downloaded
assets. Do not invent missing design details; record unknowns as gaps and stop
with BLOCKED or NEEDS_CONTEXT when required details are unavailable.
Ask a concise clarifying question whenever requirements, Figma evidence, user
intent, design details, assets, tokens, states, constraints, or implementation
choices are unclear; never proceed on unsupported assumptions.
Record this file's path, status, and unresolved gaps in
`FIGMA_DESIGN_MEMORY.md`.
